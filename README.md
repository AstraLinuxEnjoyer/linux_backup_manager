# Linux Backup Manager

Графический менеджер резервного копирования для Astra Linux 1.8.

Основной движок резервного копирования: `borgbackup`.

## Возможности

- резервное копирование выбранных каталогов;
- отдельный режим полной копии системы;
- инкрементальные копии с дедупликацией;
- максимальное сжатие через `lzma,9`;
- восстановление выбранного архива одной кнопкой;
- проверка целостности репозитория и архивов;
- журнал выполнения команд в интерфейсе;
- запуск из директории проекта через исполняемый файл `linux-backup-manager`.

## Зависимости

Для Astra Linux 1.8 установите:

```bash
sudo apt update
sudo apt install borgbackup python3-tk
```

## Запуск

```bash
chmod +x linux-backup-manager
./linux-backup-manager
```

## Как пользоваться

1. В поле `Хранилище копий` выберите отдельный диск, раздел или каталог, куда будут сохраняться резервные копии.
2. Если нужен пароль, укажите его до создания первой копии. Если пароль не нужен, оставьте поле пустым.
3. Для полного образа системы нажмите `Полная копия системы`.
4. Для копии отдельных папок добавьте их в список и нажмите `Копия выбранных каталогов`.
5. Для восстановления нажмите `Обновить список архивов`, выберите архив и каталог восстановления, затем нажмите `Восстановить выбранную копию`.

Кнопка `Подготовить хранилище` только создаёт пустое хранилище Borg. Она не создаёт резервную копию. При создании копии программа сама подготовит хранилище, если оно ещё не создано.

## Важные замечания

- Для полной копии системы запускайте программу с правами администратора:

```bash
sudo ./linux-backup-manager
```

- Не сохраняйте репозиторий резервных копий внутри каталогов, которые копируете.
- Для полного восстановления системы рекомендуется загрузиться с LiveUSB/установочного носителя, смонтировать целевой раздел и восстановить архив в точку монтирования. Восстановление прямо поверх работающей системы может не заменить занятые системные файлы корректно.
- При копировании `/` программа автоматически исключает виртуальные и временные каталоги: `/proc`, `/sys`, `/dev`, `/run`, `/tmp`, `/mnt`, `/media`, `/lost+found`.
